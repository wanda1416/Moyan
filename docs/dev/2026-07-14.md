## PyInstaller 控制台闪烁修复

- 问题：发行版启动时短暂闪烁黑色控制台窗口
- 根因：`moyan-backend.spec` 中 `console=True` 让 exe 以 Windows 控制台子系统（CONSOLE）构建，即使 Rust 侧设置 `CREATE_NO_WINDOW`，Windows 仍会先分配控制台再隐藏
- 修复：`console=True` → `console=False`，exe 改为 Windows GUI 子系统（WINDOWS）构建，从根源上不创建控制台
- stdout/stderr 已被 Rust 侧以 piped 模式捕获写入 `~/.moyan/logs/sidecar.log`，不需要控制台输出

## 图标生成修复

- 问题：安装包图标模糊，`icon.ico` 仅 1KB（应为 100KB+）
- 根因：`generate_icons.py` 使用 `append_images` 方式生成多分辨率 ICO，Pillow 存在 bug，实际只包含单个极小分辨率
- 修复：改用 `sizes` 参数直接让 Pillow 从源图缩放，生成 7 种分辨率（16/24/32/48/64/128/256px）
- 效果：`icon.ico` 从 1,046 bytes → 184,611 bytes（180KB）
- 已重新执行 `generate_icons.py` 生成正确的图标文件

## LLM 设置页激活时序修复（续）

- 问题：点击"设为默认供应商"后仍显示配置变动，关闭设置页提示保存
- 根因：`setActiveProviderId` 触发重渲染，脏检测 effect 在 `save_config` 异步完成前运行，看到旧快照
- 修复：快照更新先于 state 变更（同步更新 `llmSnapshotRef.current`，再调用 `setActiveProviderId`）
- 效果：激活供应商立即持久化，不触发 dirty 状态
