## 构建脚本 CMD 语法修复

- 修复 `build.bat` / `scripts/build-backend.bat` / `scripts/copy-sidecar.bat` / `scripts/collect-artifacts.bat` 出现“此时不应有 .`错误的问题
- 根本原因：`echo.` 在中文 Windows（代码页 936）下被误解析为“`echo .`”（回显 + 点 + 路径），CMD 报语法错误并中断脚本
- 修复措施：
  - 所有 `echo.` 替换为 `echo(`（CMD 推荐的输出空行写法，在所有代码页下都安全）
  - 4 个脚本顶部增加 `chcp 65001 >nul`，统一使用 UTF-8 代码页，避免中文文件名/路径乱码
- 间接影响修复：之前 `build-backend.bat` 报错后 `errorlevel=1` 导致 `build.bat` 提前退出，`copy-sidecar.bat` 未执行，侧车未复制到 `binaries/`。Tauri 打包时找不到侧车，NSIS 安装包只有 ~20MB（仅主程序 + NSIS 壳）；运行时侧车不存在后端启动失败
- 修正后 build 流程输出完整：PyInstaller sidecar ~216MB + 主程序 16MB + NSIS 安装包 ~218MB

## PyInstaller 从 onefile 迁移到 onedir 模式

- 根因：onefile 模式每次启动解压 ~111MB 到 `%TEMP%/_MEIxxxxx`，导致后端首次启动延迟严重
- `moyan-backend.spec`：`EXE(exclude_binaries=True)` + `COLLECT` 步骤，产物从单文件变为 `dist/moyan-backend/` 目录（exe + `_internal/`）
- `tauri.conf.json`：`externalBin` 清空，改用 `resources: ["binaries/moyan-backend/**/*"]` 捆绑整个目录
- `python_bridge.rs`：`resolve_sidecar_path()` 重写，查找 `binaries/moyan-backend/moyan-backend.exe`（Tauri v2 将 resources 放在安装目录的 `binaries/` 子目录下，而非 `resources/`）
- `copy-sidecar.bat`：从复制单文件改为 `xcopy /E` 复制整个目录结构
- `build-backend.bat`：诊断路径更新为 `dist\moyan-backend\`，新增 `_internal/` 大小输出
- `main.py`：frozen 诊断注释更新为 onedir 模式
- 效果：exe 从 111MB 降至 24MB，启动无需解压，实现秒启动；安装包从 ~218MB 降至 ~70MB+

## LLM 设置页激活与脏检测修复

- 问题 1：点击"设为默认供应商"只改了 React state，未立即持久化，用户期望激活即生效
- 修复：`handleActivate` 改为 async，立即调用 `save_config` 持久化并更新快照，显示"✓ 已切换默认供应商"
- 问题 2：保存后仍显示未保存。根因：`App.tsx` 的 `onDirtyChange` 是内联箭头函数，每次渲染重建，导致脏检测 effect 反复触发
- 修复：`App.tsx` 用 `useCallback` 稳定 `handleSettingsDirty` 和 `handleRegisterApply`；`LLMTab.tsx` 用 ref 模式隔离 `onDirtyChange`，effect 只在 `providers`/`activeProviderId` 真正变化时触发
